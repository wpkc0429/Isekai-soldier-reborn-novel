## Context

本設計文件說明全新的小說規劃如何組織。舊版「策略RPG系統面板」設定已完全棄用，改為更具現實政治感的異世界重生故事。核心差異：主角的能力來自前世靈魂記憶，而非遊戲框架賦予的數值系統。

## Goals / Non-Goals

**Goals:**
- 建立薇爾羅大陸的完整世界觀，具備內部邏輯一致性
- 主角的能力設計自然融入故事，不依賴「系統提示框」的喜劇元素
- 征服世界的動機有情感說服力（保護家人），而非純粹的野心
- 權謀佈局為核心，戰場場景服務於政治博弈，而非反過來

**Non-Goals:**
- 不保留任何遊戲化UI框架（面板、技能樹、等級條）
- 不以「爽文速食」為目標——節奏偏穩，鋪墊要有意義
- 不過度強調主角的武力碾壓，能力優勢主要體現在資訊差與預判

## Decisions

### 1. 能力框架：只有前世的生活經驗，沒有超能力

**決策**：主角沒有任何超自然能力、靈魂殘留本能或系統加成。他的全部優勢只有一個：**他活過了那八年，帶著那八年的完整記憶，重新開始。**

他在市集不被騙，是因為他見過更複雜的謊言；他能讀人，是因為他在軍中活命就靠判斷人；他睡得快，是因為老兵必須在任何地方睡著；他在事後沉默分析，是因為這是軍中的習慣，不是天賦。

**這些行為看起來「不像小孩」，但解釋很簡單：他不是小孩。**

任何一個活過八年邊境軍生涯的人，放回八歲的身體裡，都會有相同的表現。沒有神奇之處，只有經歷。

**戰鬥能力的特殊說明**：
主角在戰鬥反應上有先天的感知優勢——不是超能力，而是某些人天生具備更快的空間感知和肌肉協調性，這類人在軍中本來就存在，只是多數沒有機會被系統化培養。

前世的問題是：**有天賦，但沒有師承**。八年邊境軍只學到最基本的生存打法——怎麼用最短時間讓人倒下、怎麼在混戰裡不被後背攻擊。這是街頭技法，不是武術。

**前世後期的機緣（約21歲，石河谷之戰前一年）**：
在一次駐紮期間，主角接觸到一位流離失所的老武者，短暫習得了系統武術的基礎框架——架勢、內勁運用的原理、以及「如何讓天賦不只是本能反應」。只學了幾個月，老武者便離去，但這段記憶完整地帶進了重生。

**這個設計的結果**：
- 重生後，主角知道自己「可以學什麼」，也知道「怎麼學最快」——但8歲的身體還需要物理上的發展
- 他不是武力碾壓型，戰鬥優勢到中後期才逐漸顯現
- 核心競爭力仍是智謀和資訊差；戰鬥天賦是緊要關頭的生存保障，不是主角的招牌

**理由**：
- 移除超能力讓主角的一切行為都有普通人的根基，情感代入感更強
- 「他只是見過這些」比「他有特殊感知」更有力量——前者是任何讀者都能理解的邏輯
- 主角的限制更真實：他不知道的事情就是不知道，前世記憶有空白，記憶會出錯
- 戰鬥天賦的「未完成感」製造中期成長空間，避免主角從第一卷就是完全體

**替代方案考慮**：保留「靈魂遺留本能」的框架——但本質上是給普通行為貼上特殊標籤，捨棄。

### 2. 主角動機：保護為起點，建立根基為終點

**決策**：主角艾登的終極目標不是支配大陸，而是**建立一個任何人都動不了的根基**——自己的勢力、自己的人、能守護他們的武力與資源，然後在那個根基上娶妻生子、傳承知識，活完前世沒能活完的那段人生。

征服的規模有其上限：他不需要讓每個勢力都聽他的，他需要的是「沒有人有理由、也沒有能力對他的人動手」。兩者的差距，是野心和充分安全之間的距離。

**終點的具體樣貌**：
- **自己的勢力**：一個以他為核心、有足夠自保能力的組織/陣地——不一定有正式封號，但有實質的影響力和武力
- **家人的安全**：父親健在、妹妹過得好、身邊的人不再是別人手中的籌碼
- **婚姻與後代**：真正的家，不是管理對象
- **傳承知識**：把他在兩世人生中學到的東西——對人的判斷、對局勢的讀法、前世的歷史教訓——留給下一代，讓這些不隨他的死亡消失

**理由**：
- 終點縮小後，動機的情感重量反而更強——他要的不是天下，他要的是家
- 「我知道世界怎麼運作，但我選擇用它來護住這一小塊地方」比征服世界更有說服力
- 知識傳承的角度讓故事有跨代的重量——他的影響不只到他這輩子
- 主角在某些決策中確實需要付出道德代價，而一個更有限的目標讓這個代價更難迴避：為了「足夠安全」殺了多少人？這條線在哪裡？

---

### 3. 重生時間點：8歲，距離石河谷之戰14年

**決策**：主角重生回到8歲，石河谷之戰（前世死亡之戰）發生在他22歲，即重生後第14年。

**理由**：
- 14年的準備期足夠讓主角從零累積真實影響力
- 8歲的孩子有能力行動（不像嬰幼兒），但仍受限於成人世界，製造前中期張力
- 石河谷之戰作為「最終計時器」：主角知道那場戰爭會爆發，在那之前必須走到足夠高的位置

---

### 4. 世界觀基調：政治現實感，魔法為輔

**決策**：薇爾羅大陸以中世紀政治結構為主體，魔法存在但不普遍（約3%人口有魔力覺醒），不影響整體政治格局的「物理規則」。

**理由**：
- 讓「權謀佈局」成立需要一個規則清晰的世界，魔法過強會破壞邏輯性
- 主角前世是普通士兵（無魔力），重生後偶然覺醒小魔力，但核心競爭力仍是智謀
- 稀有魔力覺醒者作為戰略資源，可以是外交籌碼而非勝負關鍵

---

### 5. 成長路線：資訊差→財力→軍事→穩固根基

**決策**：主角的成長路線分四層，每卷聚焦一層：
1. **資訊差**（第1-2卷）：用前世記憶預判事件，建立個人信用與早期資源
2. **財力**（第2-3卷）：將資訊優勢轉化為財富，取得「能讓家人脫離危險的錢」
3. **軍事**（第3-4卷）：組建/收編忠誠的武力，有能力在石河谷之戰中保住自己的人
4. **穩固根基**（第4-5卷）：石河谷之戰後，將既有的資源、人脈、武力整合為一個能自給自足的勢力；建立家庭，將兩世的知識與教訓傳下去

**終點的定義**：不是「讓大陸按他的規則走」，而是「任何人想對他的人動手之前都要想清楚後果」。威懾，而非支配。

**理由**：
- 四層路線符合「從底層到足夠安全」的現實邏輯，避免主角跳過重要階段直接「封神」
- 終點設在「穩固根基」而非「大陸霸主」，讓主角的勝利有人的尺度，而非史詩尺度
- 知識傳承作為最後一層目標，給全書一個靜而有力的收束方式

### 8. 全書規模：100萬字，五卷分配

**決策**：全書目標總字數為 **100萬字**，以每章 2500–4000 字（平均約 3000 字）計算，約 **333 章**，分配於五卷。

**各卷章節與字數分配（建議）：**

| 卷數 | 主角年齡 | 章節數 | 估計字數 | 說明 |
|------|---------|--------|---------|------|
| 第1卷 | 8→12歲 | 約55章 | 約16.5萬字 | 建立世界感、情感鋪墊節奏偏慢 |
| 第2卷 | 12→16歲 | 約65章 | 約19.5萬字 | 進入政治視野、事件密度提升 |
| 第3卷 | 16→19歲 | 約65章 | 約19.5萬字 | 軍事佈局、多線並行 |
| 第4卷 | 19→22歲 | 約85章 | 約25.5萬字 | 高潮卷：石河谷之戰＋三線同步，篇幅最長 |
| 第5卷 | 22→25歲 | 約63章 | 約19萬字 | 收束：整合根基、建立家庭、傳承知識，節奏漸緩 |
| **合計** | | **約333章** | **約100萬字** | |

**理由**：
- 100萬字是完整獨立長篇的合理體量——足夠鋪墊十四年的成長弧線，不至於虛胖
- 第4卷最長是因為石河谷之戰本身有三條並行線，高潮章不應壓縮
- 第5卷短於第2-3卷，因為收束節奏應比建立節奏更緩——讓讀者在最後幾章真的能「坐下來」
- 章節長度維持 2500–4000 字，符合已定義的寫作節奏規範

### 9. 撰寫管理規則：字數、連貫性、人物成長追蹤

**決策**：100萬字/333章的長篇需要一套明確的管理規則，確保前後一致、人物真實成長、不出現記憶錯誤或矛盾。規則分三層。

---

#### 層一：章節字數規則

| 章節類型 | 字數範圍 | 收尾原則 |
|---------|---------|---------|
| 行動章 | 2500–3000字 | 懸而未決的問題或行動的起點 |
| 佈局章 | 3000–4000字 | 讓讀者「等著看它發揮」的期待 |
| 情感章 | 2000–3000字 | 沉靜收束，不強加懸念 |
| 高潮章 | 不設上限 | 場景必須完整，不在緊要處截斷 |

**硬性規則**：
- 單章不得低於2000字（情感章除外不強制，但過短需有理由）
- 單章一般不超過5000字——超過代表場景可以拆開
- 每章有明確的「進入狀態」和「離開狀態」：章末的世界和章首的世界有差別

---

#### 層二：劇情連貫性規則

333章跨越14年，需要一套記錄系統防止矛盾。以下四份文件在撰寫過程中持續維護：

**① 事件日誌**（每章末更新）
記錄格式：`[章節號] [發生了什麼] [誰知道] [後果尚未展開的項目]`
- 目的：查找「第X章說過什麼」，避免在第200章和第50章互相矛盾

**② 未兌現伏筆清單**（每章末更新）
記錄已埋下但尚未兌現的伏筆，標記「預計在哪一卷/哪個節點揭露」
- 規則：每卷結束前，清理本卷埋下的伏筆，未兌現者移入下卷

**③ 已改變的時間線節點**
記錄主角的行動導致「和前世不同」的事件
- 格式：`[原本會發生什麼] → [現在實際發生什麼] → [連鎖影響]`
- 目的：主角的前世記憶只在「尚未改變的節點」上有效，此表區分哪些預判還可用

**④ 資訊不對稱表**（每卷末更新）
追蹤「誰知道什麼」——這在一部以資訊差為核心的故事中至關重要
- 對每個重要角色記錄：知道主角真實計劃的哪些部分？知道哪些前世秘密？
- 規則：主角不應「忘記」某個角色已知道某件事

---

#### 層三：人物成長追蹤

**更新頻率**：每卷結束後更新一次所有核心角色的狀態卡；重大轉折（死亡、背叛、新同盟）發生後立即更新，不等到卷末。

**每個角色的狀態卡追蹤以下項目：**

| 項目 | 說明 |
|------|------|
| 年齡 | 本卷結束時的年齡 |
| 身體狀態 | 健康、傷病、重要的身體變化 |
| 能力水準 | 戰鬥等級、技能、知識的本卷進展 |
| 關係狀態 | 與主角及其他核心人物的信任程度（用簡單描述，非數值） |
| 知情程度 | 知道主角真實計劃的哪些部分（重要：防止人物「神奇地知道」或「莫名地不知道」） |
| 心理狀態 | 本卷的成長、創傷、核心衝突 |
| 位置與角色 | 在故事中的地理位置與當前扮演的角色 |

**規則**：
- 人物不能在沒有理由的情況下「回到之前的狀態」——成長是不可逆的，創傷是真實的
- 前世戰友（塔利、賽門、艾拉・布蘭）的成長弧線要與主角的計劃**同步但獨立**——他們在成長，不只是工具

#### 層四：新登場人物登記系統

333章的故事會陸續出現數十個甚至上百個有名字的角色。需要一套分級登記制度，確保每個人物首次登場的資訊被記錄，後續再出現時有據可查。

**人物分級：**

| 等級 | 定義 | 管理方式 |
|------|------|---------|
| A 級（核心人物） | 預先規劃、貫穿全書的角色（艾登、梅拉、塔利等9人） | 每卷末更新完整狀態卡（層三） |
| B 級（重要配角） | 在故事中出現3次以上、對主角計劃有實質影響 | 首次登場時建立簡化卡，每次重要改變後更新 |
| C 級（功能性角色） | 出現1–2次，完成特定場景功能後退場 | 首次登場時登記基本資訊，結案後標記「已退場」 |

**登記規則：**
- 任何有名字、有台詞、或對劇情有直接影響的角色，**必須在該章完成後登記**，不得事後補記
- 登記在一份持續維護的「人物名冊」（character-registry）中，按首次登場章節排序
- 若一個 C 級角色在後續章節意外變得重要，**立即升級為 B 級**並補充資料

**新人物首次登場登記格式（B 級）：**
```
姓名：[全名 / 別名]
首次登場：第X章
等級：B
身份：[職業、社會地位、與主角的關係]
外貌：[一行，讓後續章節保持一致]
性格要點：[2–3個核心特質]
與主角的關係：[第一次見面的印象 / 已知的利害關係]
知情程度：[知道主角哪些事？]
當前狀態：[存活 / 死亡 / 位置 / 角色]
備注：[任何可能造成連貫性問題的細節]
```

**C 級登記格式（精簡）：**
```
姓名：[全名]
首次登場：第X章
等級：C
身份：[一行]
作用：[在這個場景做了什麼]
外貌：[若有特徵描述則記錄]
狀態：[存活 / 退場 / 死亡]
```

**升降級機制：**
- C → B：當該角色在後續章節再次出現且有劇情功能，立即補全 B 級欄位
- B → A：當該角色被確定為全書重要人物，移入核心狀態卡（層三）管理

---

**理由**：
- 100萬字如果沒有管理系統，矛盾積累到後期無法修補
- 字數規則讓每章有形狀，不因填字數而膨脹
- 資訊不對稱表對「主角靠資訊差布局」的故事尤其關鍵——一旦搞混「誰知道什麼」，整個謀算就失去邏輯
- 人物名冊讓「三章前出現過的商人」不會在第200章突然外貌和名字對不上

---

### 10. 撰寫流程輔助：大綱、節奏、時間軸、AI工作階段

**決策**：除了管理文件（Decision #9）之外，撰寫流程本身也需要幾個固定機制，確保333章的長篇能持續推進、不失方向、不失節奏。

---

#### 10-A：卷前章節大綱

**每卷開始前，必須先完成本卷的章節級大綱。**

每行一章，格式：
```
第X章 [暫定標題] | [章節類型] | [場景] | [在場人物] | [本章目的] | [預計字數]
```

範例：
```
第8章 碼頭的下工時間 | 行動章 | 王都碼頭 | 艾登、塔利 | 第一次找到塔利，測試他是否可用 | 2800字
```

**規則**：
- 大綱不必完美，可以在寫作中修改，但開始寫之前必須有大綱
- 大綱一行代表對這章的「承諾」——若最後與大綱偏差太多，需回頭確認是否影響後續章節

---

#### 10-B：章節前置清單

**每章開始前確認以下項目（2分鐘，不跳過）：**

- [ ] 上一章的「離開狀態」是什麼？本章是否從那個狀態繼續？
- [ ] 本章涉及哪些角色？他們目前的狀態卡是否已確認？
- [ ] 有沒有未兌現的伏筆應該在這章推進或兌現？
- [ ] 本章類型是什麼（行動／佈局／情感）？字數目標？
- [ ] 這章結束時，讀者應該感受到什麼？（不是「發生了什麼」，是「感受是什麼」）

---

#### 10-C：情感節奏追蹤

用三個標記記錄每章完成後的情感溫度，維護一份簡單的節奏日誌：

| 標記 | 含義 |
|------|------|
| ▲ | 緊張／高壓／衝突 |
| → | 推進／布局／穩步 |
| ▼ | 沉靜／情感／落地 |

**節奏規則**：
- 連續4章以上 ▲ → 必須插入至少1章 ▼
- 連續3章以上 ▼ → 讀者可能失去緊張感，考慮推進節奏

#### 10-D：故事內部日曆

14年跨度的故事需要明確的時間座標，維護一份「故事當前時間」記錄：

記錄格式：`[章節號] 故事時間：[年份/季節/月份] | 主角年齡：X歲 | 距石河谷之戰：X年`

**規則**：
- 每次發生超過2週的時間跳躍，必須在章節中明確標示或在時間記錄中標注
- 季節影響場景描寫，不能出現「春天種的作物，下一章就到秋收，但主角還在同一個場合」的矛盾
- 前世記憶的「預判」要與當前故事時間對照——某些事件只在特定時間點成立

#### 10-E：AI撰寫工作階段模板

每次開始新的撰寫 session，提供以下「本次簡報」讓 Claude 快速建立 context：

```
## 本次撰寫簡報

**目標章節**：第X章（第N卷）
**故事時間**：[年份季節]，主角X歲，距石河谷之戰X年
**章節類型**：[行動／佈局／情感]，目標字數：XXXX字

**上章摘要**（3行以內）：
[發生了什麼，以什麼狀態結束]

**本章涉及人物及當前狀態**：
- [人名]：[現在在哪、知道什麼、與主角的關係]
- [人名]：...

**本章目的**：[用一句話說這章要完成什麼]

**需要注意的連貫性事項**：
- [有沒有前章埋下的伏筆需要在本章推進？]
- [有沒有人物上章說過的話本章需要呼應？]
```

---

**理由**：
- 卷前大綱是對自己的承諾，有路線圖才不會在第30章突然不知道往哪走
- 章節前置清單把「寫作」和「管理」分開——清單在寫之前做，不在寫的過程中打斷自己
- 情感節奏是讀者體驗的心跳，333章如果全程相同頻率會讓讀者麻木
- 故事內部日曆對14年跨度的故事是基礎設施，不記錄就會在時間上出錯
- AI撰寫簡報是這個具體使用情境的關鍵——每次session讓Claude立刻知道「現在在哪裡、要寫什麼」，不需要重新解釋整個故事

## Risks / Trade-offs

| 風險 | 緩解策略 |
|------|---------|
| 主角過於冷靜，讀者難以情感投入 | 以家人場景為感情錨點，定期拉回主角的人性面 |
| 前世記憶的資訊優勢讓劇情失去張力 | 前世是士兵非貴族，對高層政治的認知有限，必須靠觀察補足 |
| 目標縮小後故事缺乏史詩感 | 史詩感來自代價的重量，而非目標的規模——「為了讓家人安全殺了多少人」比「征服世界」更沉 |
| 14年準備期前半段節奏過慢 | 第1卷聚焦幾個高密度事件，非線性時間跳躍呈現成長 |

## Open Questions

- 主角是否在重生後不久就有小規模的魔力覺醒，還是維持到後期才覺醒？
- 前世的某位戰友是否會在故事中以「尚未成為士兵的孩童」身份再次出現？
- 反派（推動石河谷之戰的貴族）是主角幼年就認識的人，還是完全陌生的勢力？

### 7. 人物群像：青梅竹馬 + 夥伴團隊

**決策**：在家人之外，主角圍繞著兩個人際圈成長——「從小一起長大的人」與「未來因目標走到一起的戰友」。兩個圈子的加入動機和關係質地完全不同，製造豐富的人際張力。

**青梅竹馬（梅拉）的設計原則：**
- 前世艾登14歲就去當兵，沒有真正活過正常的童年人際關係；本世他第一次擁有「一個從小認識的人」
- 她不是主角計劃的一部分，而是計劃之外的存在——唯一一個讓主角有時忘記自己在佈局的人
- 她直接、不怕說真話，是少數當面說「你有點奇怪」的人，也是少數主角不會對她進行「評估」的人

**夥伴團隊的設計原則：**
- 每個人都有自己加入的理由，不是被主角「招募」，而是因為某個時刻他們的利益或信念與主角的方向一致
- 每個人代表一種主角自身不具備的能力（武力/分析/滲透/貴族知識/街頭智慧）
- 隨著故事推進，夥伴對主角的理解逐漸加深，但沒有人知道全貌——包括青梅竹馬

**理由**：
- 征服路線的主角若只有「工具型人際關係」，長期閱讀會疲乏；青梅竹馬是情感的呼吸口
- 夥伴各有背景與動機，讓隊伍本身的人際關係成為劇情的一條線
