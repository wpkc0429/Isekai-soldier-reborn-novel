## ADDED Requirements

### Requirement: 敘事視角與基調
採用**第一人稱限定視角（主角視角）**，偶爾穿插旁觀視角。

**第一人稱主角視角（90%）：**
- 讀者跟著主角的思維，包含前世記憶的運用、對當前局勢的分析、以及偶爾閃過的個人情感
- 前世記憶以「斜體」標示，與當下思維明確區分
- 前世記憶不是「回憶插敘」，而是主角在判斷當下時自然浮現的參照：*前世這個人死在自己挖的坑裡。這世也差不多。*

**第三人稱旁觀視角（10%）：**
- 使用場景：展示主角不在場的敵方謀算、或需要讀者知道主角不知道的資訊
- 製造資訊差張力：「讀者知道有埋伏，主角不知道」——拉緊節奏

**整體基調**：沉穩、有重量。不用輕佻的語氣，幽默只在人物關係中自然浮現（非刻意設計的笑點）。主角的思維是老兵的思維：直接評估、不廢話、偶爾說一句冷的話讓氣氛停頓。

#### Scenario: 前世記憶的呈現方式
- **WHEN** 主角面對一個即將做出錯誤決定的對手
- **THEN** 不以大段回憶插敘呈現，而是一行斜體：*前世這個部署被帝國側翼撕開。這世的人還沒學到。* 然後主角繼續行動。

### Requirement: 章節節奏規範
每章以2500-4000字為標準，無固定三段式，根據內容決定結構。

**節奏類型：**

- **行動章**（2500-3000字）：推進劇情，強調效率，結尾留下「現在怎麼辦」的問題
- **佈局章**（3000-4000字）：主角謀算展開，需要足夠篇幅讓讀者跟上邏輯，結尾留下「等著看它發揮」的期待
- **情感章**（2000-3000字）：家人場景、個人時刻，節奏最慢，不刻意推劇情，結尾沉靜而非懸念

**高潮前後的節奏處理：**
- 高潮前：連續短章（行動章），剪短每章的收尾，製造速度感
- 高潮章本身：不限字數，確保場景完整
- 高潮後：至少一章「情感章」讓讀者落地，主角的個人反應比戰略分析更重要

**每卷的節奏分配（建議）：**
- 開卷：慢（讓讀者感受到這個世界）
- 中段：標準，偶爾加速
- 高潮前：快
- 結尾：慢

#### Scenario: 章節結尾的鉤子設計
- **WHEN** 行動章結束
- **THEN** 鉤子是一個懸而未決的問題或一個行動的開始，而非完整收束。不用「主角成功了真棒」收尾，用「現在，第二步開始了」。

### Requirement: 對話規範與人物聲音
**比例：對話+行動 60% / 思維+描寫 40%**

這是一部節奏偏沉的小說，比例比通常爽文保留更多的敘事空間，但依然以對話作為主驅動力。

**各人物的說話方式：**

| 人物 | 說話特點 |
|------|---------|
| 主角艾登 | 話少、精準，不補充不必要的解釋；沉默本身也是回答 |
| 父親羅爾 | 口語、樸實、偶爾說一句意外地準確的話（農民哲學） |
| 母親艾拉 | 碎念但每句都有資訊量；偶爾說出讓主角停頓的話 |
| 妹妹莉亞 | 直接、活潑，說話不繞彎子，大人以為她不懂，她其實懂一半 |
| 塔利 | 粗豪、直白，說話帶點自嘲；對主角有時忍不住說一句刺話，但行動永遠跟上 |
| 安塔公爵 | 永遠客氣，永遠有餘地，說話讓人找不到直接反駁的點 |

**對話書寫原則：**
- 同一段對話不超過4-5個來回沒有動作描寫的間隔（防止「乒乓對話」)
- 主角的沉默是武器——讓對方多說
- 不給人物寫超過3行的獨白演說，有就分段或打斷

### Requirement: 主角內心的呈現方式
主角沒有「系統框架」，內心世界的呈現更接近一個有豐富前世經驗的人的思維流。

**三種思維模式，以不同方式呈現：**

**戰術評估（一行到三行，冷靜）：**
費格站在左側，覆蓋範圍是門到窗口。若要離開，右側的走廊是唯一選項——但右廊有人聲，此刻不是時機。

**前世記憶（斜體，簡短）：**
*前世石河谷之戰前三天，安塔公爵也是這個表情。那次他在算的是誰死了對他最有利。*

**個人情感（罕見，但出現時分量最重）：**
不是「主角感到了什麼」的直述，而是行為和細節：主角把手放在妹妹的頭上，停了很久。沒有說話，也沒有必要說話。

**禁忌：**
- 不用大段解說「主角為什麼這樣決定」——決定在行動中展現，原因在後續揭曉
- 不用感嘆號表達主角的情緒（他是一個死過一次的人）
- 不刻意安排「讓讀者爽的時刻」——讓局勢自然收束，爽感從合理性中生長

#### Scenario: 主角做出艱難決定後的呈現
- **WHEN** 主角決定犧牲一個無辜的局外人來保全更大的佈局
- **THEN** 不用一段「主角內心痛苦的告白」；而是：主角做完了，走到窗邊，看了很久外面。然後轉身說了下一步要做的事。讀者在這個停頓裡感受到代價——比任何獨白都更重。

### Requirement: 章節命名規範
每章標題簡潔，以「動作/狀態」或「名詞」形式呈現，不劇透，但看完後回頭一想「早就說了」。

**格式**：`第X章　標題`（無副標）

**範例：**
- `第1章　重生這天下起了雨`
- `第7章　父親的那條路`
- `第14章　無名的商人`
- `第20章　石河谷以北`
- `第31章　世界按我的規則`

**命名原則：**
- 不超過10個字
- 不說明「主角做了什麼了不起的事」
- 帶有空間感、時間感、或人物感
- 好的章節標題讀起來像一個場景的快照，不是一個事件的摘要

#### Scenario: 標題的伏筆感
- **WHEN** 第一卷末的最後一章，父親活過了那個時間點
- **THEN** 章節標題可以是「第20章　黃昏和平常一樣」——平淡，但讀者知道這個平淡來之不易。標題本身不說破。

### Requirement: 全書規模與撰寫管理

**全書規模**：目標100萬字，五卷約333章，各卷篇幅依故事密度分配：

| 卷數 | 章節數 | 估計字數 |
|------|--------|---------|
| 第1卷 | 約55章 | 約16.5萬字 |
| 第2卷 | 約65章 | 約19.5萬字 |
| 第3卷 | 約65章 | 約19.5萬字 |
| 第4卷 | 約85章 | 約25.5萬字（高潮卷） |
| 第5卷 | 約63章 | 約19萬字 |

**撰寫管理系統**：
100萬字的長篇需要配套的追蹤文件，詳見 `writing-management` spec，包含：
- 事件日誌、未兌現伏筆清單、已改變時間線節點、資訊不對稱表
- 人物成長狀態卡（核心A級）、人物名冊（B/C級新登場角色）
- 卷前章節大綱、章節前置清單、情感節奏追蹤、故事內部日曆
- AI撰寫工作階段簡報模板
