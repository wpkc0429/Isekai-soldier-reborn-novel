# 事件日誌模板

## 用途

記錄故事中發生的重要事件，以及其知情者和未來應兌現的後果。
防止「前面說過的事後來忘記跟進」。

---

## 格式

```
## 事件日誌

| 事件ID | 章節 | 事件描述 | 知情者 | 待兌現後果 | 兌現狀態 |
|--------|------|---------|-------|-----------|---------|
| E001   | 1-7  | 父親以車軸問題拒絕致命包車 | 父親、主角 | 德文貴族的結局（確認父親逃過一劫） | ✓ 兌現 1-8章 |
| E002   | 1-25 | 主角第一次接觸塔利，說「半年後左肩會壞」 | 主角、塔利 | 塔利確認左肩問題被避免 | ✓ 兌現 1-30章附近 |
| E003   | 2-8  | 主角讓賽門「意外」得到更好機會 | 主角（賽門不知道） | 賽門察覺有人在幫他，之後確認是主角 | ✓ 兌現 2-卷 |
| E004   | 3-55 | 主角對陳卡說「前次那樣的結果」，立刻修正 | 主角、陳卡 | 陳卡可能繼續想這句話的意思 | 待觀察 |
| E005   | （待填）| （描述事件）| （知情者列表）| （需要跟進的後果）| 未兌現 |
```

---

## 欄位說明

| 欄位 | 說明 |
|------|------|
| 事件ID | 唯一識別碼，格式E001, E002...按順序 |
| 章節 | 發生的卷-章（例：1-7 代表第1卷第7章） |
| 事件描述 | 30字以內，說明發生了什麼 |
| 知情者 | 哪些角色知道這件事（可能和讀者知道的不同） |
| 待兌現後果 | 這件事在未來需要跟進的部分 |
| 兌現狀態 | 未兌現 / ✓兌現（記錄兌現章節） |

---

## 使用規則

1. **每章寫完後更新**：在每章完成後，立刻記錄本章的新事件
2. **每卷結束前審查**：確認所有「未兌現」的事件是否有計劃處理
3. **跨卷的未兌現事件要特別標注**：如果某事件的兌現在下一卷，用 `→卷X` 標注
