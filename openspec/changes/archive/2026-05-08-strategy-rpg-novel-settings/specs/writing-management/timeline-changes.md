# 已改變時間線節點表

## 用途

記錄本世主角的行動如何改變了前世的命運，以及這些改變帶來的連鎖影響。
這份表確保：當主角的干預導致後果時，可以追蹤改變的因果鏈。

---

## 格式

```
| 節點ID | 前世原本事件 | 本世實際結果 | 改變方式 | 連鎖影響 | 章節 |
|--------|------------|------------|---------|---------|------|
```

---

## 欄位說明

| 欄位 | 說明 |
|------|------|
| 節點ID | 格式T001, T002...按時間順序 |
| 前世原本事件 | 如果主角不干預，前世的結局 |
| 本世實際結果 | 主角干預後實際發生的事 |
| 改變方式 | 主角用什麼方式改變（信息干預/財務操作/武力/政治操縱） |
| 連鎖影響 | 這個改變帶來的後續效應 |
| 章節 | 干預發生的章節 |

## 使用規則

1. **每次干預後立刻記錄**：重大的時間線改變在發生的章節後立刻更新
2. **連鎖影響要持續追蹤**：「連鎖影響」欄位在後來的章節中可能需要補充更多層次
3. **如果改變沒有成功**：也要記錄「嘗試改變但失敗」的節點——主角不是每次都成功
