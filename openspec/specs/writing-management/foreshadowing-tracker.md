# 未兌現伏筆清單模板

## 用途

追蹤已埋設但尚未揭露的伏筆，確保每一個伏筆都有計劃的揭露時間。

---

## 格式

```
## 未兌現伏筆清單

| 伏筆ID | 埋設章節 | 伏筆內容（讀者看到什麼） | 預計揭露 | 最遲揭露章 | 揭露後意義 | 狀態 |
|--------|---------|---------------------|---------|-----------|----------|------|
| F001   | 1-5     | 父親說起包車任務，主角「胃下沉」 | 1-8 | 1-8 | 父親命運轉折 | ✓已揭露 |
| F002   | 1-1     | 主角死時只想到家人的瑣事 | →卷5 | 5-末 | 知識傳承的動機根源 | 未揭露 |
```

---

## 欄位說明

| 欄位 | 說明 |
|------|------|
| 伏筆ID | 格式F001, F002...對應foreshadowing-list.md中的ID |
| 埋設章節 | 卷-章格式 |
| 伏筆內容 | 讀者讀到時看見的是什麼（不是「真相」） |
| 預計揭露 | 計劃在哪一卷哪一章揭露，格式：卷-章或→卷X（跨卷） |
| **最遲揭露章** | **硬性截止點**——寫到該章前未揭露 → 進入 alert 清單。對「累積式」伏筆（F013/F016 類）填「累積（無單一截止）」 |
| 揭露後意義 | 揭露後讀者應該理解到什麼 |
| 狀態 | 未揭露 / ✓已揭露（記錄揭露章節） |

---

## Scope 註記：本檔案的「知情者」相關欄位

> 本檔案若有「誰知道這個伏筆」相關欄位，記錄的是**單次伏筆的埋設者 / 知曉者**（per-foreshadowing），與 `info-asymmetry-table.md` 的「該角色對主角的整體知道程度」是不同層次。本檔案不被 SSOT 規則涵蓋，不需改 link。

---

## Alert 規則（2026-05-10 新增，依 P0-3）

- **觸發條件**：寫到「當前章 + 10」時，掃描所有未揭露伏筆，若「最遲揭露章 ≤ 當前章 + 10」→ 進入下一輪 brief 的「需揭露伏筆」欄
- **掃描時機**：`chapter-sync.md` §B（每 10 章 full sync）執行時自動跑
- **未及時揭露的處理**：超過最遲揭露章仍未揭露 → 標 `[OVERDUE]`；必須在下一個 sync 內決定「立即補揭露 / 重新指派最遲章 / 取消伏筆」三選一

---

## 使用規則

1. **埋設時立刻記錄**：每次埋設伏筆，立刻在此表新增一行
2. **揭露前確認**：在揭露伏筆前，回來確認這個伏筆的預計意義，避免揭露後效果弱
3. **每卷結束前審查**：確認是否有「應該在本卷揭露但忘記的伏筆」
4. **不要同時有太多未揭露伏筆**：建議在任意時間點，未揭露的核心伏筆不超過7個
