## ADDED Requirements

### Requirement: 章節字數規則

| 章節類型 | 字數範圍 | 收尾原則 |
|---------|---------|---------|
| 行動章 | 2500–3000字 | 懸而未決的問題或行動的起點 |
| 佈局章 | 3000–4000字 | 讓讀者「等著看它發揮」的期待 |
| 情感章 | 2000–3000字 | 沉靜收束，不強加懸念 |
| 高潮章 | 不設上限 | 場景必須完整，不在緊要處截斷 |

**硬性規則：**
- 單章不得低於2000字（情感章可例外，但需有理由）
- 單章一般不超過5000字——超過代表場景可以拆開
- 每章有明確的「進入狀態」和「離開狀態」：章末的世界與章首的世界有差別

---

### Requirement: 事件日誌

每章完成後更新。記錄格式：

```
[章節號] [故事時間] [發生了什麼] [誰知道] [待兌現的後果]
```

**用途**：查找「第X章說過什麼」，防止第200章與第50章互相矛盾。

**規則**：
- 每章完成後當場更新，不事後補記
- 「待兌現的後果」欄位若為空，代表這個事件已完全閉合

---

### Requirement: 未兌現伏筆清單

記錄已埋下但尚未兌現的伏筆。記錄格式：

```
[伏筆描述] | 埋設章節：第X章 | 預計兌現：第N卷 | 狀態：待兌現／已兌現
```

**規則**：
- 每卷結束前清理本卷埋下的伏筆，未兌現者移入下卷並標注原因
- 跨越兩卷以上仍未兌現的伏筆需要主動確認：是刻意延遲還是遺忘？

---

### Requirement: 已改變時間線節點

主角的行動導致「和前世不同」的事件，是判斷前世記憶是否仍有效的依據。記錄格式：

```
[原本前世會發生什麼] → [本世實際發生什麼] → [連鎖影響] | 記錄於第X章
```

**規則**：
- 主角依賴某個前世記憶做出預判後，若結果與預期不符，必須在此表記錄修正
- 此表是主角「前世知識的有效區域」邊界圖——已改變的節點之後，前世記憶僅供參考

---

### Requirement: 資訊不對稱表

追蹤「誰知道什麼」。每卷末更新一次。格式：

```
[角色姓名]
- 知道主角的真實計劃：[知道哪些部分]
- 知道主角的前世記憶：[知道／不知道／懷疑]
- 對主角的信任程度：[描述，非數值]
- 本卷新增的知情內容：[這卷讓他知道了什麼]
```

**規則**：
- 主角不應「忘記」某個角色已知道某件事而再次洩露
- 若某角色「神奇地知道」某件事卻沒有記錄來源，這是連貫性錯誤

---

### Requirement: 核心人物狀態卡（A 級）

適用於：艾登、梅拉、塔利、賽門、艾拉・布蘭、凱斯、尤拉、羅爾・莫爾、莉亞・莫爾。

**更新頻率**：每卷末更新；重大轉折（死亡、背叛、新同盟）後立即更新。

每個角色追蹤項目：

| 項目 | 說明 |
|------|------|
| 年齡 | 本卷結束時的年齡 |
| 身體狀態 | 健康、傷病、重要的身體變化 |
| 能力水準 | 戰鬥等級、技能、知識的本卷進展 |
| 關係狀態 | 與主角及其他核心人物的信任程度 |
| 知情程度 | 知道主角真實計劃的哪些部分 |
| 心理狀態 | 本卷的成長、創傷、核心衝突 |
| 位置與角色 | 在故事中的地理位置與當前扮演的角色 |

**規則**：
- 人物不能在沒有理由的情況下「回到之前的狀態」——成長是不可逆的
- 前世戰友的成長弧線要與主角的計劃**同步但獨立**——他們在成長，不只是工具

---

### Requirement: 人物名冊（B/C 級新登場角色）

**分級定義：**

| 等級 | 定義 |
|------|------|
| B 級 | 出現3次以上、對主角計劃有實質影響的配角 |
| C 級 | 出現1–2次、完成特定場景功能後退場的功能性角色 |

**登記規則**：
- 任何有名字、有台詞、或對劇情有直接影響的角色，**必須在該章完成後登記**
- C 級角色意外變得重要時，立即升級為 B 級並補充資料

**B 級登記格式：**
```
姓名：[全名 / 別名]
首次登場：第X章
等級：B
身份：[職業、社會地位、與主角的關係]
外貌：[一行，確保後續章節一致]
性格要點：[2–3個核心特質]
與主角的關係：[第一次見面的印象 / 已知的利害關係]
知情程度：[知道主角哪些事]
當前狀態：[存活 / 死亡 / 位置 / 角色]
備注：[可能造成連貫性問題的細節]
```

**C 級登記格式：**
```
姓名：[全名]
首次登場：第X章
等級：C
身份：[一行]
作用：[在這個場景做了什麼]
外貌：[若有特徵描述則記錄]
狀態：[存活 / 退場 / 死亡]
```

### Requirement: 卷前章節大綱

**每卷開始前，必須先完成本卷的章節級大綱。**

每行一章，格式：
```
第X章 [暫定標題] | [章節類型] | [場景] | [在場人物] | [本章目的] | [預計字數]
```

**規則**：
- 開始寫之前必須有大綱，大綱可在寫作中修改
- 若最後與大綱偏差過大，需確認是否影響後續章節

### Requirement: 情感節奏追蹤

用三個標記記錄每章完成後的情感溫度：

| 標記 | 含義 |
|------|------|
| ▲ | 緊張／高壓／衝突 |
| → | 推進／布局／穩步 |
| ▼ | 沉靜／情感／落地 |

**節奏規則**：
- 連續4章以上 ▲ → 必須插入至少1章 ▼
- 連續3章以上 ▼ → 考慮推進節奏
- 各卷整體節奏：開卷→、中段▲→交替、高潮前▲▲、高潮▲▲▲、收束▼

---

### Requirement: 故事內部日曆

維護故事當前時間的連續記錄。格式：

```
[章節號] 故事時間：[年份/季節/月份] | 主角年齡：X歲 | 距石河谷之戰：X年
```

**規則**：
- 超過2週的時間跳躍，必須在章節中標示或在此記錄中標注
- 季節影響場景描寫，不能出現季節矛盾
- 前世記憶的「預判」要與當前故事時間對照——某些事件只在特定時間點成立
