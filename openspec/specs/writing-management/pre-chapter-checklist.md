# 章節前置清單

## 用途

每章開始寫作之前，花5分鐘確認這7個問題。
防止「寫完才發現設定衝突」或「章節目的不清楚」。

**確認 6 與確認 7 是硬性檢查**——若任一為「否」，本章必須調整或重寫，不能寫出去。

---

## 五項確認

### 確認1：這章的人物狀態是否正確？

在本章開始前，確認以下問題：
- 本章出現的每個人，他們的年齡是否正確（參考各卷基準表）？
- 他們的力量等級在這個時間點是否符合設定？
- 他們對主角的「知道程度」是否符合 info-asymmetry-table.md 中的記錄？

**確認方式**：
```
本章人物：XXX, XXX, XXX
確認年齡：是 / 否（如否，修正至：）
確認知道程度：是 / 否（如否，修正至：）
```

---

### 確認2：本章有沒有違反前面已建立的事實？

在開始寫前，回想：
- 本章是否涉及某個在前面章節已有說明的地點/事件/物品？
- 如果涉及，確認描述一致

**常見衝突類型**：
- 距離（A地到B地要幾天，前面說的和現在寫的是否一致）
- 人物關係（某人和某人是否在這個時間點認識了）
- 力量等級（某人的武力水準是否符合當前章節的時間點）

---

### 確認3：本章的目的是什麼？

這章結束後，讀者應該得到什麼？

從以下三個中選一個主要目的（允許有次要目的）：
- **信息**：讀者/主角得到了一個重要的信息或判斷
- **情感**：讀者對某個人物的情感連結加深
- **局勢**：故事世界的某個局勢發生了改變

**填寫格式**：
```
主要目的：（信息/情感/局勢）
具體內容：（一句話說明讀者得到了什麼）
```

---

### 確認4：本章是否需要兌現某個伏筆或記錄新事件？

查閱：
- foreshadowing-tracker.md：是否有預計在本章或本章附近揭露的伏筆？
- event-log-template.md：本章是否有新的重要事件需要記錄？

**確認格式**：
```
需揭露伏筆：無 / （伏筆ID）
需記錄新事件：無 / （事件描述）
```

---

### 確認5：本章的末尾是否有「往前帶」的元素？

每章末尾應該讓讀者想繼續讀。

「往前帶」的方式：
- **懸念**：留下一個問題（不是刻意的懸崖式，而是自然的「接下來會怎樣」）
- **期待**：某個場景或對話讓讀者期待下一章的到來
- **情緒餘韻**：某個情感場景在結尾留下一個沉默的空間

**確認格式**：
```
結尾方式：懸念 / 期待 / 情緒餘韻
具體計劃：（一句話說明結尾的最後一個意象或問題）
```

---

### 確認6【硬性】：本章是否有外部場景，而不是整章主角內心？

第一卷後半卷出現過多次「整章為主角內心獨白／清單式自我盤點」的章節（ch033、ch053、ch055），違反 `internal-monologue-guide.md` 與 CLAUDE.md「不讓主角做大段的自我解釋式內心獨白」的禁令。

**硬性標準**：本章必須有至少一個外部場景——對白、動作、環境感知、其他人在場——且這個外部場景必須佔本章正文 50% 以上。

**確認格式**：
```
外部場景：（一句話描述場景，誰在哪裡做什麼）
場景占比：≥50% ✓ / <50%（如不足，必須補場景再寫）
```

**若違反**：本章必須先設計一個外部場景才能下筆。內心活動可以有，但必須鑲嵌在場景中，不能反過來。

---

### 確認7【硬性】：結尾最後一個意象是否帶有懸念、期待或情緒餘韻？

不只是「想到下一步」，是讀者讀完最後一句後想翻下一頁。

**禁止的結尾類型**：
- 純內心宣告作結（「現在開始了，往下走就好了」這類封閉式收束）
- 純情緒收束無外部信號（卷末 ▼ 收束時，最後一章必須在尾段 1–3 句內補一個外部信號：信件、外部來訪、可見的環境變化）
- 「接下來他繼續做事」這類弱收束

**確認格式**：
```
結尾最後一個意象：（具體寫出最後 1–3 句的內容方向）
帶有的元素：懸念 / 期待 / 情緒餘韻（必須至少一項）
是否為內心獨白式封閉收束：否 ✓ / 是（如為「是」，必須重寫結尾）
```

## 卷末 sync 必查清單（卷末同步用）

每卷最後一章寫完後（不只是首次同步，是每次卷末），逐項對照以下項目，確認管理/設定文件與正文事實一致。第一卷的 vol1-writing §7 因為缺少這份清單，留下了多處未同步的尾巴；第二卷起每卷結束都應跑這份清單。

| 項目 | 對照來源（正文） | 對照目標（設定/管理） | 動作 |
|------|---------------|------------------|------|
| 各人物在卷末的實際年齡 | 各章開頭資訊區塊 + 卷時間跨度 | `character-state-cards.md` 卷末欄、`family-cards.md` 各卷命運欄、`protagonist-settings/character-card.md` | 互相對照、修正 |
| 各人物對主角的「知道程度」 | 卷內所有對話與獨白實際說出的內容 | `info-asymmetry-table.md` 卷末更新欄 | 以正文為準同步 |
| 卷內新埋設與已揭露的伏筆 | `event-log` 已記錄事件、章節內容 | `foreshadowing-tracker.md`、`foreshadowing-list.md` | 補入遺漏；移除重複編號；更新揭露狀態 |
| 卷末主角的力量等級與隱藏狀態 | 覺醒章節與後續章節 | `character-state-cards.md` 主角欄、`info-asymmetry-table.md` 主角武力欄 | 確認其他人的「知道程度」與此一致 |
| 卷末改變的時間線節點 | 主角的所有干預章節 | `timeline-changes.md` | 補入新節點 |

## 快速版（熟悉後使用）

```
章節：第X卷第X章「（標題）」

1. 人物狀態：✓ / 需確認（XXX）
2. 事實一致：✓ / 需確認（XXX）
3. 章節目的：（一句話）
4. 伏筆/事件：無 / （說明）
5. 結尾計劃：（一句話）
6.【硬】外部場景占比 ≥50%：✓ / 否（必須補場景）
7.【硬】結尾非內心獨白封閉收束：✓ / 否（必須重寫結尾）
```
