# 章節前置清單

## 用途

每章開始寫作之前，花5分鐘確認這5個問題。
防止「寫完才發現設定衝突」或「章節目的不清楚」。

---

## 五項確認

### 確認1：這章的人物狀態是否正確？

在本章開始前，確認以下問題：
- 本章出現的每個人，他們的年齡是否正確（參考各卷基準表）？
- 他們的力量等級在這個時間點是否符合設定？
- 他們對主角的「知道程度」是否符合 info-asymmetry-table.md 中的記錄？

**確認方式**：
```
本章人物：XXX, XXX, XXX
確認年齡：是 / 否（如否，修正至：）
確認知道程度：是 / 否（如否，修正至：）
```

---

### 確認2：本章有沒有違反前面已建立的事實？

在開始寫前，回想：
- 本章是否涉及某個在前面章節已有說明的地點/事件/物品？
- 如果涉及，確認描述一致

**常見衝突類型**：
- 距離（A地到B地要幾天，前面說的和現在寫的是否一致）
- 人物關係（某人和某人是否在這個時間點認識了）
- 力量等級（某人的武力水準是否符合當前章節的時間點）

---

### 確認3：本章的目的是什麼？

這章結束後，讀者應該得到什麼？

從以下三個中選一個主要目的（允許有次要目的）：
- **信息**：讀者/主角得到了一個重要的信息或判斷
- **情感**：讀者對某個人物的情感連結加深
- **局勢**：故事世界的某個局勢發生了改變

**填寫格式**：
```
主要目的：（信息/情感/局勢）
具體內容：（一句話說明讀者得到了什麼）
```

---

### 確認4：本章是否需要兌現某個伏筆或記錄新事件？

查閱：
- foreshadowing-tracker.md：是否有預計在本章或本章附近揭露的伏筆？
- event-log-template.md：本章是否有新的重要事件需要記錄？

**確認格式**：
```
需揭露伏筆：無 / （伏筆ID）
需記錄新事件：無 / （事件描述）
```

---

### 確認5：本章的末尾是否有「往前帶」的元素？

每章末尾應該讓讀者想繼續讀。

「往前帶」的方式：
- **懸念**：留下一個問題（不是刻意的懸崖式，而是自然的「接下來會怎樣」）
- **期待**：某個場景或對話讓讀者期待下一章的到來
- **情緒餘韻**：某個情感場景在結尾留下一個沉默的空間

**確認格式**：
```
結尾方式：懸念 / 期待 / 情緒餘韻
具體計劃：（一句話說明結尾的最後一個意象或問題）
```

---

## 快速版（熟悉後使用）

```
章節：第X卷第X章「（標題）」

1. 人物狀態：✓ / 需確認（XXX）
2. 事實一致：✓ / 需確認（XXX）
3. 章節目的：（一句話）
4. 伏筆/事件：無 / （說明）
5. 結尾計劃：（一句話）
```
